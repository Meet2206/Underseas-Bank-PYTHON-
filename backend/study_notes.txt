## Tech Stack

| Layer | Technology | Purpose |
|---|---|---|
| **Backend Framework** | FastAPI | Modern async API framework with auto Swagger docs |
| **Database** | PostgreSQL | ACID-compliant relational DB for financial data |
| **ORM** | SQLAlchemy | Python ORM for database interactions |
| **Cache / Broker** | Redis | OTP storage (encrypted, with TTL), rate limiting, caching, Celery broker |
| **Background Tasks** | Celery | Async jobs (emails, notifications, interest calculations) |
| **Reverse Proxy** | Nginx | SSL termination, load balancing, reverse proxy |
| **Auth** | JWT + bcrypt | Token-based auth with password hashing |
| **OTP Security** | `cryptography` lib | Encrypt OTPs before storing in Redis |

---

## Architecture

```
Client (Web/Mobile)
    ↓
Nginx (Reverse Proxy / SSL / Load Balancing)
    ↓
FastAPI (API Layer)
    ↓
Auth Layer (JWT + OTP)
    ↓
Service Layer (Business Logic)
    ↓
SQLAlchemy ORM
    ↓
PostgreSQL
    ↕
Redis (OTP cache, rate limiting, analytics cache)
    ↕
Celery Worker (emails, background tasks)
```

## Core Modules

### 1. User / Customer Management
- Registration, Login/Logout, KYC details, Profile management
- Fields: name, email, phone, address, Aadhaar/ID, password (hashed)

### 2. Account Management
- Create bank account, view details, multiple accounts per user
- Fields: account_number, account_type (Savings/Current), balance, status

### 3. Transactions
- Deposit, Withdraw, Transfer between accounts, Transaction history
- Fields: transaction_id, sender_account, receiver_account, amount, type, status, timestamp

### 4. Balance & Statement
- Check balance, view last N transactions, download statement

### 5. Security
- Password hashing (bcrypt), JWT auth, OTP verification, Rate limiting, Input validation

### 6. Admin Panel
- View all users, freeze/unfreeze accounts, view suspicious transactions, generate reports

---

## Advanced Modules

### 7. OTP Verification
- **Flow**: User action → Generate OTP → Encrypt with `cryptography` → Store in Redis (with TTL) → Send via Celery email task → User submits OTP → Decrypt → Verify

### 8. Email Notifications (via Celery)
- Account creation, transfer alerts, loan EMI reminders, credit card bill alerts
- Uses `smtplib` or SendGrid/Mailgun, dispatched as async Celery tasks

### 9. Interest Calculations
- Savings: `Interest = (Balance × Rate × Time) / 100`
- Can run as scheduled Celery task (daily/monthly cron)

### 10. Loan Module
- Types: Personal, Home, Education, Vehicle
- **EMI Formula**: `EMI = [P × R × (1+R)^N] / [(1+R)^N – 1]`
- Fields: loan_type, loan_amount, interest_rate, duration_months, monthly_emi, remaining_balance, status

### 11. Fixed Deposits (FD)
- Lock money for a period at a fixed rate
- Fields: amount, interest_rate, start_date, maturity_date, status

### 12. Credit Card System
- Credit limit tracking, spending, monthly bill, minimum payment, late penalty
- Fields: credit_limit, used_amount, available_limit, billing_date, due_date

### 13. Expense Analytics Dashboard
- Category-wise spending analysis: Food, Essentials, Transport, Shopping, Entertainment, Bills, Travel, Healthcare
- **Pie chart** data for frontend (Chart.js)
- Auto-categorization via merchant name keyword matching
- APIs: `/analytics/monthly-expenses`, `/analytics/spending-trend`, `/analytics/top-categories`

### 14. Financial Health Score (0–100)
| Factor | Weight | Logic |
|---|---|---|
| Savings Ratio | Savings / Income | Higher = better |
| Expense Distribution | Balanced vs. luxury-heavy | Balanced = better |
| Credit Utilization | Used / Limit | Lower = better |
| EMI Burden | EMI / Income | Lower = better |
| Late Payments | Count | Fewer = better |

> 90–100: Excellent · 70–89: Good · 50–69: Average · < 40: Critical

### 15. AI Financial Tips (Rule-based)
- "Your savings are low. Try saving at least 20% of your income."
- "Your credit card usage is high. Consider paying some balance."
- "Food expenses are high this month."
- "Your entertainment spending increased by 40%."

## Key Workflows

| Flow | Steps |
|---|---|
| **Login** | Request → Nginx → FastAPI → OTP Generate → Encrypt → Redis Store → Celery Send Email → User Verifies → Decrypt → JWT Issued |
| **Transfer** | Request → Balance Check → Record Transaction → Update Balances → Celery Send Alert Email |
| **Analytics** | Request → Multi-table fetch → Aggregate → Cache in Redis → Return Dashboard Data |

## Why PostgreSQL over MySQL/MongoDB?

- **ACID compliance** — critical for financial transactions
- **Reliable transaction handling** — BEGIN/COMMIT/ROLLBACK
- **Advanced data integrity** — foreign keys, constraints
- **Better concurrency** — MVCC (Multi-Version Concurrency Control)
- MongoDB's eventual consistency is dangerous for financial ledgers
